Commercial scope proposal

Delivery plan for the MWU Fluency Pipeline

The research workflow runs from Phase I to Phase V. The commercial delivery should be separated into a small Validation Sprint and layered delivery tiers, so each commitment has a clear acceptance gate, dependency boundary, and risk profile.

Recommended path

Start with the monologue benchmark, then fund the highest-labor annotation layer, then add research analysis and final reporting only after definitions and gold standards are signed off.

Pre-layerVS

Validation Sprint - monologue benchmark

Purpose: prove that the software can reproduce the provided expert Praat/TextGrid/Excel baseline before committing to the real multilogue pipeline.

Covers

  • Phase II gold replay
  • Phase III transcript split test
  • Phase V matrix skeleton
  • Net-new local WebUI validation console

Deliverables

  • Input manifest and method log
  • 0.25 / 0.35 threshold artifacts
  • RAW and TIDY transcript files
  • Validation report and matrix skeleton

Acceptance

  • Pause count matches exactly
  • Key durations match within 0.001 s
  • 0.25 SpeakerX matrix matches baseline
  • 0.35 clearly marked as no-gold

Boundary

  • Monologue only
  • No diarization validation
  • No final lexical/MWU analysis
  • Deductible only within agreed window

Workbook check: the supplied SpeakerX workbook has no separate 0.35 threshold gold columns; 0.35 is generated for schema and process verification only until the research team supplies a matching gold baseline.

Layer 1aL1a

AI diarization and speaker isolation

Purpose: process real multilogue recordings by proposing speaker turns and producing per-speaker muted-mirror audio for later Praat processing.

Covers

  • Phase I
  • Configurable speaker count
  • AI speaker diarization
  • Muted-mirror WAV generation

Deliverables

  • Diarization JSON
  • Per-speaker audio files
  • Review TextGrid draft
  • Correction-rate report

Acceptance

  • Measured against client gold cuts
  • Speaker error and edit rate reported
  • Overlaps and uncertainty surfaced
  • Human correction remains expected

Risk

  • Probabilistic AI component
  • Sprint does not validate this layer
  • Depends on real multilogue samples
  • Overlapping speech may require manual correction
Layer 1bL1b

Deterministic Praat automation

Purpose: reduce the most repetitive Praat pause-boundary and duration-calculation work with configurable, reviewable, Praat-compatible outputs.

Covers

  • Phase II core
  • Configurable pause thresholds
  • Praat CLI / TextGrid generation
  • Duration calculations

Deliverables

  • Threshold-specific TextGrids
  • Segment duration tables
  • Review-ready artifacts
  • Parameter and version logs

Acceptance

  • Full unsegmented timeline preserved
  • Gold-sample count and duration error reported
  • Parameters adjustable without code changes
  • Manual Praat review remains final

Risk

  • Deterministic and controllable
  • Calculation chain tested in Sprint
  • Does not depend on cloud ASR
  • Still needs human review of difficult acoustic cases
Layer 2L2

Research analysis expansion

Purpose: add the transcript, AS-unit, pause-location, lexical and MWU analysis needed to answer the research questions, once definitions are stable.

Covers

  • Phase III
  • Phase IV
  • Early Phase V integration
  • Optional word-level alignment add-on

Deliverables

  • RAW-TIMING and TIDY-PHRASE files
  • AS-unit / clause mapping tables
  • Lexical and MWU feature tables
  • Pause-location features

Start condition

  • AS-unit rules signed off
  • MWU definition signed off
  • Syllable / repair coding rules signed off
  • Gold examples supplied

MFA boundary

  • Base scope uses reviewed silence and clause boundaries
  • MFA adds word-level timing precision
  • MFA is separately quoted and validated
  • No word-level claim without alignment evidence

Final matrix and reporting system

Purpose: turn reviewed outputs and research features into a stable final workbook, codebook, validation package and reporting workflow.

  • Cross-phase synthesis

  • Reporting workflow

Deliverables

  • Analysis matrix CSV/XLSX
  • Codebook and method log
  • Validation dashboard/report

Acceptance

  • Schema signed off
  • Gold validation reported
  • Traceability from source to metric
  • Publication/archive package complete

Boundary

  • Statistical modeling is separate
  • Final research interpretation remains researcher-owned
  • Requires stable upstream outputs
